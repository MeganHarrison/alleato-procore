# Alleato OS - Master Implementation Plans

This document serves as the central index for all implementation plans and architectural documentation for the Alleato OS project. Plans are organized by initiative in separate folders.

## Overview

Alleato OS is a modern alternative to Procore (construction project management software) built with:
- **Frontend**: Next.js 15 with App Router, Supabase, OpenAI ChatKit UI, ShadCN UI, and Tailwind CSS
- **Backend**: Supabase (PostgreSQL with RLS, Auth, Storage)
- **AI System**: OpenAI Agents SDK and Codex MCP
- **Analysis Tools**: Playwright-based screenshot capture and AI analysis

## Project Philosophy

The Alleato OS implementation follows these core principles:

1. **UI-First Approach** - Build the component system first, then layer on schema-driven functionality
2. **Self-Consistency** - All pages use shared component library for unified UX
3. **Extendability** - New modules use the same underlying system
4. **Autonomous Execution** - Plans detailed enough for coding agents to implement with minimal clarification
5. **Traceability** - Every UI element maps to a component; every feature has tests

3. Follow the implementation checklist step-by-step
4. Update the plan document with progress and learnings

## Creating New Plans

### Folder Structure

When creating a new plan initiative:

1. **Create a new folder** under `docs/plans/` with a descriptive name (kebab-case)
2. **Add a README.md** in that folder explaining the initiative
3. **Create plan documents** with the `plans-` prefix
4. **Update this index** to link to the new initiative folder

### Plan Document Structure

Each plan should follow this structure:

1. **Purpose** - What this plan covers
2. **Status** - Current completion state
3. **Implementation Checklist** - Actionable items with checkboxes
4. **Evidence** - Screenshots, test results, artifacts
5. **Notes & Learnings** - Decisions made, patterns discovered
6. **Related Plans** - Cross-references to other documents

### Naming Convention

- Folders: `initiative-name` (kebab-case)
- Plan files: `plans-feature-name.md` (kebab-case with `plans-` prefix)
- Examples:
  - `budget-implementation/plans-budget-tree-view.md`
  - `meetings-implementation/plans-fireflies-integration.md`

## Maintenance Guidelines

### Updating Plans

- Keep plans current as implementation progresses
- Add new sections for significant features or refactors
- Archive completed plans to initiative folder's `/archive/` subdirectory
- Update this index when adding or removing initiatives

### Plan Document Standards

- Use markdown checkboxes for tracking: `- [ ]` or `- [x]`
- Include code examples where applicable
- Link to related files using relative paths
- Add timestamps to status updates
- Include screenshots in plan's `/screenshots/` subdirectory if large
